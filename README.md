# ПАПКА OOP-PROJECT: Система управления задачами

### Обзор OOP-Project

Это расширенный учебный проект, демонстрирующий:
- Управление сложными объектами через TaskManager
- Численные методы решения СЛАУ
- Анализ численной устойчивости
- Сериализацию в JSON

## Основные компоненты

### **1. Task.h / Task.cpp** — Базовый класс задачи

```cpp
enum class TaskStatus {
    PENDING,    // Задача ожидает выполнения
    RUNNING,    // Задача выполняется
    COMPLETED,  // Задача завершена успешно
    FAILED      // Задача не удалась
};
```

**Основные методы:**
- `get_status_string()` — получить статус в виде строки
- `to_json()` / `from_json()` — сериализация в JSON

### **2. MyTask.h / MyTask.cpp** — Решение СЛАУ

**Назначение:** Решает системы линейных уравнений **Ax = b** двумя методами.

**Входные параметры (JSON):**
```json
{
  "method": "gauss" или "lu",
  "matrix": [[2, 1], [1, 3]],
  "b": [8, 13]
}
```

**Методы решения:**
- `solveGauss()` — Метод Гаусса с выбором ведущего элемента
- `solveLU()` — LU-разложение через Eigen

**Результаты:**
- Решение системы
- Число обусловленности
- Анализ чувствительности к возмущениям

### **3. TaskManager.h / TaskManager.cpp** — Управление задачами

**Функциональность:**
- Добавление/удаление задач
- Выполнение задач
- Фильтрация по статусу
- Сохранение/загрузка JSON
- Статистика

## Технические особенности OOP-Project

**Используемые библиотеки:**
- **Eigen** — линейная алгебра
- **nlohmann/json** — JSON сериализация

**Ключевые алгоритмы:**

#### Метод Гаусса с частичным выбором ведущего элемента
- Выбирает максимальный элемент для исключения
- Более устойчив к ошибкам округления
- Сложность: O(n³)

#### LU-разложение
- Разложение A = LU (L — нижнетреугольная, U — верхнетреугольная)
- Используется библиотека Eigen::PartialPivLU
- Сложность: O(n³)

#### Число обусловленности
```
cond(A) ≈ ||A||₁ · ||A⁻¹||₁
```
- Мера чувствительности системы к возмущениям
- Низкое значение → хорошо обусловленная система
- Высокое значение (>1e6) → плохо обусловленная система
