### Обзор проекта

Это учебный проект на C++, который демонстрирует принципы объектно-ориентированного программирования через систему управления задачами (Task Manager). Проект включает решение систем линейных уравнений двумя различными методами с анализом численной устойчивости.

---

## Структура папки `project`

### **Главный компонент: OOP-Project**

Проект состоит из следующих файлов:

#### **1. Task.h / Task.cpp** — Базовый класс задачи
```cpp
enum class TaskStatus {
    PENDING,    // Задача ожидает выполнения
    RUNNING,    // Задача выполняется
    COMPLETED,  // Задача завершена успешно
    FAILED      // Задача не удалась
};
```

**Основные методы класса Task:**
- Конструктор: инициализирует ID, описание и временные метки
- `get_status_string()` — получить статус в виде строки
- `to_json()` / `from_json()` — сериализация в JSON формат
- Логирование действий в векторе `log`

---

#### **2. MyTask.h / MyTask.cpp** — Специализированный класс для решения систем уравнений

**Назначение:** Решает системы линейных уравнений **Ax = b** двумя методами.

**Входные параметры (передаются в JSON):**
```json
{
  "method": "gauss" или "lu",
  "matrix": [[n×n матрица коэффицие��тов]],
  "b": [вектор правой части из n элементов]
}
```

**Основной функционал:**

| Метод | Описание |
|-------|---------|
| `solveGauss()` | Метод Гаусса с частичным выбором ведущего элемента |
| `solveLU()` | LU-разложение (используется библиотека Eigen) |
| `computeConditionNumber()` | Вычисляет число обусловленности матрицы |
| `execute()` | Основной метод выполнения задачи |
| `print_result()` | Выводит результаты в красивом формате |

**Результаты выполнения:**
- Решение системы `A | b`
- Число обусловленности `cond(A)` — мера численной устойчивости
- Чувствительность решения — как изменяется решение при малых возмущениях входных данных
- Логи выполнения

---

#### **3. TaskManager.h / TaskManager.cpp** — Менеджер задач

**Функциональность:**
- Создание и удаление задач
- Выполнение задач (одной или всех сразу)
- Фильтрация по статусу
- Сохранение и загрузка из JSON файла
- Статистика по выполненным задачам

**Основные методы:**
```cpp
add_task(Task)           // Добавить новую задачу
run_task_by_id(id)       // Выполнить задачу по ID
run_all_pending()        // Выполнить все ожидающие задачи
get_tasks_by_status()    // Получить задачи определённого статуса
save_to_file(filename)   // Сохранить в JSON
load_from_file(filename) // Загрузить из JSON
show_statistics()        // Вывести статистику
```

---

#### **4. OOP-Project.cpp** — Главная программа с интерактивным меню

**Меню приложения:**
```
1. Добавить новую задачу
2. Показать все задачи
3. Показать задачи по статусу

6. Показать статистику
7. Сохранить задачи в файл
```

**Функции для работы с пользователем:**
- `menuAddTask()` — добавление новой задачи с параметрами
- `menuShowAll()` — вывод всех задач
- `menuShowByStatus()` — фильтрация по статусу
- `menuRunTask()` / `menuRunAll()` — выполнение задач
- `menuSave()` / `menuLoad()` — работа с файлами
- `printMenu()` — отображение меню

## Технические особенности

### **Используемые библиотеки:**
- **Eigen** — линейная алгебра и матричные операции
- **nlohmann/json** — работа с JSON формата

### **Ключевые алгоритмы:**

#### **Метод Гаусса с частичным выбором ведущего элемента**
- Более устойчив к ошибкам округления
- Выбирает максимальный элемент для исключения
- Сложность: O(n³)

#### **LU-разложение**
- Разложение матрицы A = LU
- L — нижнетреугольная, U — верхнетреугольная
- Решение через прямую и обратную подстановку
- Сложность: O(n³)

#### **Число обусловленности**
```
cond(A) ≈ ||A||₁ · ||A⁻¹||₁
```
- Мера чувствительности системы к возмущениям
- Низкое значение → хорошо обусловленная система
- Высокое значение → плохо обусловленная система

## Пример использования

```cpp
// 1. Создание менеджера
TaskManager tm;

// 2. Подготовка параметров
nlohmann::json params = {
    {"method", "gauss"},
    {"matrix", {{2.0, 1.0}, {1.0, 3.0}}},
    {"b", {8.0, 13.0}}
};

// 3. Создание и добавление задачи
MyTask task(1, "Solve 2x + y = 8, x + 3y = 13", params);
tm.add_task(task);

// 4. Выполнение
tm.run_task_by_id(1);

// 5. Сохранение
tm.save_to_file("results.json");
```

- JSON сериализация
